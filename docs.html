Bienvenido a la documentación del microframework TinyApp (1.2.0) el cuál tiene control de rutas, middlewares, conexión a base de datos mysql y oracle, configuración para api restful, integración de librerías externas y archivos assets (frontend), helpers, validador de formularios, subida de archivos, manejo de sesiones; y arquitectura para manejo de vistas, modelos y controladores.

Nota: Este framework fue creado por Héctor de Jesús Coello Gómez con la intención de ayudar en el desarrollo de proyectos web pequeños, simples y funcionales.
Primeros pasos

Una vez descargado el framework debes configurarlo creando un archivo .env en la carpeta raíz del proyecto. Para crear el archivo .env debes hacerlo creando una copia del archivo de ejemplo que se encuentra en el proyecto: env-example.txt (este archivo una vez copiado puedes eliminarlo).

Tu primer configuración en el archivo .env debe ser la URL de tu proyecto, lo harás modificando la propiedad BASE_URL.

STATUS_PROJECT=dev BASE_URL=http://localhost/tinyapp/ ...

Al poner el proyecto en producción debes cambiar el valor de STATUS_PROJECT de "dev" a "pro" en el archivo .env

STATUS_PROJECT=pro BASE_URL=https://www.mywebsite.com ...

Si deseas realizar pruebas con el modelo App que viene de prueba en el proyecto, deberás crear la siguiente base de datos:

CREATE DATABASE tinyapp; USE tinyapp; CREATE TABLE `users` ( `id` int(5) PRIMARY KEY AUTO_INCREMENT, `fullname` VARCHAR(500) NOT NULL, `status` TINYINT(1) NOT NULL DEFAULT 1 ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci; INSERT INTO `users` (`fullname`) VALUES ('Elizabeth Thompson'), ('George Williams'), ('Mary Jones'), ('David Smith');


Ahora puedes crear tus controladores.

Controladores

Debes crear tus controladores con el namespace App\Controllers. Se pueden obtener los datos enviados por GET, POST o parámetro URL, con la función input(). Se sugiere nombrar a los controladores en formato CamelCase con la palabra Controllers al final.

/app/controllers/HomeController.php


namespace App\Controllers; class HomeController { $username = input('username'); // GET or POST or URL Param }

Para el uso de controladores en API Rest, se debe extender al controlador de "Response", para poder dar respuesta tipo JSON, como se ve a continuación.

namespace App\Controllers; use Core\Response; class StudentApiController extends Response { public function index(){ $students = array( array('id' => 100, "name" => 'Mary Jones' ), array('id' => 101, "name" => 'David Williams' ), array('id' => 102, "name" => 'Matt Smith' ), array('id' => 103, "name" => 'Cris Brown' ) ); $data = array( 'success' => true, 'data' => $students, 'message' => "" ); return $this->json($data, 200); } }

En el tercer parámetro del método $this->json() se establece el nombre de la configuración de la cabecera (header) de la respuesta, por defecto utiliza la configuración "default" en $api_config en el archivo de configuración del proyecto app/Config.php

Rutas

Rutas tipo GET, POST, PUT y DELETE.

/app/routes.php

require_once './app/controllers/HomeController.php'; ... // Rutas Route::register("get", "/", [ HomeController::class ], 'index' ], [ 'test' ] ); Route::register("post", "/login", [ HomeController::class, 'login' ] ); // Rutas con parámetros Route::register("get", "/student/{id}", [ StudentController::class, 'show' ] ); Route::register("delete", "/student", [ StudentController::class, 'destroy' ] ); Route::register("put", "/student/{id}/status/{status}", [ StudentController::class, 'update' ] ); // Rutas para API Route::register("get", "/api/student", [ StudentApiController::class, 'index' ] );
Vistas

Las vistas se debe llamar en los controladores con el método View::get().

namespace App\Controllers; use Core\Validator; use Core\File; class HomeController { public function index(){ // Vistas siendo llamadas View::get('partials/header'); View::get('home'); View::get('partials/footer'); // Llamadas con parámetros $data = array( 'app' => "TinyApp", 'message' => $message, ); View::get('home', $data); } }
Modelos

Debes crear tus modelos con el namespace App\Models, como se ve a continuación.

namespace App\Models; class App { public function index(){ ... } }

Para usar este modelo en un controlador, debes hacer un require_once del archivo require_once './app/models/App.php'; y usarlo con use App\Models\App.

Base de datos

Para conectarte a la base de datos debes usar la librería Core\Databases\DB, como se ve a continuación.

namespace App\Models; use Core\Databases\DB; class User { public function single($id){ $db = DB::getInstance(); return $db->db_select_row("SELECT * FROM users WHERE id=?", [ $id ]); } public function list(){ $db = DB::getInstance(); return $db->db_select("SELECT * FROM users"); } public function list_by_status($status){ $db = DB::getInstance(); return $db->db_select("SELECT * FROM users WHERE status=?", [ $status ]); } public function insert($name){ $db = DB::getInstance(); return $db->db_insert_lastid("INSERT INTO users (fullname) VALUES (?)", [ $name ]); } public function update($id, $name){ $db = DB::getInstance(); return $db->db_update("UPDATE users SET fullname = ? WHERE id = ?", [ $name, $id ]); } public function delete($id){ $db = DB::getInstance(); return $db->db_delete("DELETE from users WHERE id = ?", [ $id ]); } }

La configuración de la base de datos se realiza en el archivo app/Config.php

Middlewares

Los middlewares se pueden implementar en el tercer párametro del método Route::register(), en el archivo de rutas app/routes.php.

require_once './app/controllers/HomeController.php'; ... // Middleware "test" Route::register("get", "/", HomeController::class, 'index' , [ 'test' ] );

Los archivos middlewares se deben crear en la carpeta app/middlewares/, por ejemplo app/middlewares/Test.php, el archivo debe tener el siguiente formato:

namespace App\Middlewares; class Test { public function run() { // Acá deben ir el código propio del middleware // Puedes usar validación de sesiones, tokens, redireción... return true; } }
Validaciones

Para usar las validaciones en los controladores se debe implementar use Core\Validator;.

namespace App\Controllers; use Core\Validator; class HomeController { public function index(){ Validator::check([ [ 'user', [ 'string' ] ], [ 'email', [ 'email', 'min(8)' ] ], ]); ... } }

El primer parámetro del método Validator::check() recibe un arreglo de dos dimensiones. La primer dimensión tiene el nombre del valor que se está recibiendo (input). La segunda dimensión es un arreglo con las validaciones aplicables a ese valor.

Las validaciones aplicables son las siguientes: string, numeric, integer, float, boolean, email, file, max(5), min(4), val(1, 0), allows_null.

El método Validator::check() recibe un segundo parámetro, este es tipo booleano y sirve para habilitar o deshabilitar la redireción automática en la validación. Por defecto es true.

Sesiones

Si deseas manejar sesiones en tu proyecto, debes activarlo desde el archivo .env poniendo true en LOAD_SESSION y agregando una clave/id de sesión personalizada.

LOAD_SESSION=true SESSION_ID=clav3_personaliz4d4
use Core\Session; // Crear una sesión $session = new Session(); $data_session = array( "id"=> 100, "username"=> 'Hector'); $session->create($data_session); // Cambiar datos de sesión $session = new Session(); $session->set_data("username", "Hector Coello"); // Obtener datos de sesión $session = new Session(); $id = $session->get_data("id"); // Validar sesión $session = new Session(); $validate = $session->validate(); $validate = $session->validate("login"); // Eliminar sesión $session->delete();
Almacenamiento

Para guardar archivos se debe usar la librería Core\File, de igual manera si se desea validar el envío de la imagen, se puede validar con el tipo 'file', como se ve a continuación.

namespace App\Controllers; use Core\Validator; use Core\File; class HomeController { public function index(){ Validator::check([ [ 'photo', [ 'file' ] ], ]); $upload = File::upload( 'photo', 'photos', [ 'max_size' => 2000, 'allowed_types' => "image/jpeg|pdf" ]); // Optional if(!$upload['status']){ set_errors(array($upload['message'])); back(); } ... } }

La configuración para habilitar el almacenamiento de archivos ($load_upload_files) se realiza en app/Config.php


Helpers

Los Helpers son funciones globales que se deben agregar en el archivo app/helpers.php. Para que los Helpers estén habilitados, se deben precargar ($load_helper = true) desde el archivo de configuración app/Config.php

public function format_email(){ ... }

Para usar este modelo en un controlador, debes hacer un require_once del archivo ( require_once './app/models/App.php'; ) y usarlo con use App\Models\App.


Librerías
